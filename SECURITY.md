# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 5.x     | ✅ Yes    |
| < 5.0   | ❌ No     |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Use GitHub's private vulnerability reporting instead: go to this repository's
**Security** tab → **Report a vulnerability**. This opens a private advisory
visible only to maintainers until a fix is ready.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

We will respond within 72 hours and aim to release a fix within 14 days.

## Scope

This tool is designed for analyzing files you own or have permission to analyze.
Do not use it against files you do not have authorization to inspect.
